Nightwatch Agent — How It Works

An AI first responder that investigates production incidents automatically

1

Your Application Runs on AWS

A real service runs in a container, processing requests and logging everything. Business as usual.

ECS Fargate — runs the application
2

Something Goes Wrong

The service starts failing. Errors appear in the logs. Customers are affected.

CloudWatch Logs — captures all application logs
3

AWS Detects the Problem AUTOMATIC

CloudWatch monitors error rates continuously. When errors exceed the threshold, an alarm fires and sends an alert.

CloudWatch Alarm → SNS — detects and routes the alert
4

Nightwatch Agent Wakes Up AUTOMATIC

The alert automatically triggers a small program that wakes up the AI agent and says: "There's a problem. Investigate."

Lambda (Trigger) — receives the alert, starts the agent
5

AI Investigates Like a Senior Engineer

The AI agent (powered by Amazon Bedrock with Claude) decides what data to check. It reads service health, error logs, performance metrics, and deployment history — all from real AWS data.

Bedrock Agent (Claude Sonnet) — the AI brain that reasons
Lambda (Tools) — reads real data from CloudWatch and ECS
6

Agent Connects the Dots

The AI correlates signals across all data sources: "These errors point to the payment service timing out. CPU and memory are fine — this is not our code, it's an upstream dependency."

7

Report Delivered to Your Team

The agent sends a clear, actionable report to Slack: what broke, why, what evidence supports it, and what the engineer should do next.

Slack — delivers the investigation report
DynamoDB — saves the investigation record
8

Engineer Makes the Decision

The on-call engineer reads the report — root cause, evidence, recommendation — and decides what to do. The agent investigated in 60 seconds what would take a human 45 minutes.
